# Q46: Kernel panic: how do you debug?

## In-depth Explanation (Nvidia Interview Style)

- Use kernel logs (`dmesg`), crash dumps, and stack traces to identify the cause.
- Reproduce the panic with debug options enabled.
- Use tools like `crash`, `kgdb`, and kernel debugging configs.

### Interview Tips
- Be ready to discuss panic handlers, oops messages, and post-mortem analysis.
